# Mini-Agent 子模块管理手册

本文档说明 bjt_agent 项目中 Mini-Agent 子模块的管理规范。

## 概述

bjt_agent 使用 Mini-Agent 作为子模块，托管在 `https://github.com/Zhangdong1977/Mini-Agent.git`。

## 克隆项目

首次克隆需要同时初始化子模块：

```bash
git clone https://github.com/Zhangdong1977/bjt_agent.git
cd bjt_agent
git submodule init
git submodule update
```

或者使用 `--recurse-submodules` 一步到位：

```bash
git clone --recurse-submodules https://github.com/Zhangdong1977/bjt_agent.git
```

## 同步子模块更新

当主项目提示子模块有更新时：

```bash
git pull
git submodule update --remote Mini-Agent
```

## 修改子模块

如果你需要修改 Mini-Agent 子模块，按以下步骤操作：

### 1. 在子模块中提交并推送

```bash
cd Mini-Agent
git add .
git commit -m "your fix description"
git push origin main
```

### 2. 更新主项目的子模块引用

```bash
cd ..
git add Mini-Agent
git commit -m "chore: update Mini-Agent to latest"
git push origin master
```

### 3. 通知其他开发者

告知团队成员执行同步命令：

```bash
git pull
git submodule update --remote Mini-Agent
```

## 子模块常用命令

| 命令 | 作用 |
|------|------|
| `git submodule status` | 查看子模块当前 commit |
| `git submodule sync` | 同步 .gitmodules 中的 URL 到本地 |
| `git submodule update --remote` | 拉取子模块远程最新 |
| `git submodule update --force` | 强制同步到主项目记录的版本 |

## 注意事项

1. **子模块的改动需要两次提交**：一次在子模块内部，一次在主项目更新指针
2. **提交前先拉取**：在子模块中提交前，先 `git pull origin main` 确保最新
3. **主项目记录的是 commit 而非分支**：这确保所有开发者使用相同版本

## 常见问题

**Q: 子模块显示 "修改的内容" 但不知如何处理？**

```bash
cd Mini-Agent
git checkout HEAD -- .
git status
```

**Q: 如何切换到其他分支？**

```bash
cd Mini-Agent
git checkout <branch-name>
cd ..
git add Mini-Agent
git commit -m "chore: switch Mini-Agent to branch-name"
```

**Q: 子模块 URL 变了怎么办？**

```bash
git pull
git submodule sync
git submodule update --remote
```
