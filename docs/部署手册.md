## 1. 系统架构概览

```
┌─────────────────────────────────────────────────────────────────────┐
│                           用户浏览器                                  │
└─────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼ HTTPS (:443)
┌─────────────────────────────────────────────────────────────────────┐
│                           Nginx 反向代理                              │
│                     (端口 3000/8000 → 后端服务)                       │
└─────────────────────────────────────────────────────────────────────┘
                                    │
                    ┌───────────────┼───────────────┐
                    ▼               ▼               ▼
            ┌────────────┐ ┌────────────┐ ┌────────────┐
            │   前端 Vue3  │ │  后端 FastAPI │ │ RAG Memory  │
            │   :3000     │ │    :8000     │ │   :3001     │
            └────────────┘ └────────────┘ └────────────┘
                    │               │               │
                    └───────────────┴───────────────┘
                                    │
                    ┌───────────────┴───────────────┐
                    ▼                               ▼
            ┌────────────┐                   ┌────────────┐
            │  Celery    │                   │  PostgreSQL │
            │  Workers   │                   │    :5432    │
            │ (review/   │                   └────────────┘
            │  parser)   │
            └────────────┘
                    │
                    ▼
            ┌────────────┐
            │    Redis   │
            │  :6379     │
            │ (Broker +  │
            │   SSE)     │
            └────────────┘
```

### 服务说明

| 服务 | 端口 | 说明 |
|------|------|------|
| **Frontend** | 3000 | Vue3 前端，用户界面 |
| **Backend API** | 8000 | FastAPI 后端，核心业务逻辑 |
| **RAG Memory** | 3001 | 企业知识库检索服务 |
| **Celery Review** | - | 标书审查异步任务 Worker |
| **Celery Parser** | - | 文档解析异步任务 Worker |
| **PostgreSQL** | 5432 | 数据持久化 |
| **Redis** | 6379 | Celery Broker + SSE 消息 |

## 4. 方式一：直接部署（开发/测试环境）

### 4.1 系统用户创建（推荐）

```bash
# 创建专用用户
sudo useradd -m -s /bin/bash bjt
sudo usermod -aG docker bjt  # 如使用 Docker

# 切换到部署用户
sudo su - bjt
```

### 4.2 安装基础依赖

```bash
# 安装系统依赖
sudo apt update && sudo apt upgrade -y
sudo apt install -y git lsof curl wget

# 安装 Conda
cd /tmp
wget https://repo.anaconda.com/miniconda/Miniconda3-latest-Linux-x86_64.sh
bash Miniconda3-latest-Linux-x86_64.sh -b -p ~/miniconda3
rm Miniconda3-latest-Linux-x86_64.sh

# 添加 Conda 到 PATH（可选，写入 ~/.bashrc）
echo 'export PATH="$HOME/miniconda3/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc
```

### 4.3 创建 Python 环境

```bash
# 使用 py311 环境（必须）
conda create -n py311 python=3.11 -y
conda activate py311

# 验证 Python 版本
python --version  # 应显示 3.11.x
```

### 4.4 安装 Node.js

```bash
# 使用 nvm 安装 Node.js 18+
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
source ~/.bashrc
nvm install 18
nvm use 18
node --version  # 应显示 18.x.x
npm --version   # 应显示 9.x.x
```

### 4.5 获取代码

```bash
# 克隆代码仓库（使用部署者自己的 GitHub 账号）
cd ~
git clone https://github.com/Zhangdong1977/bjt_agent.git
cd bjt_agent

# 初始化 Git 子模块
git submodule update --init --recursive
```

### 4.6 安装后端依赖

```bash
conda activate py311
cd bjt_agent/backend

# 安装核心依赖
pip install -r requirements.txt

# 验证关键包安装
python -c "import fastapi, celery, sqlalchemy, asyncpg; print('All packages OK')"
```

### 4.7 安装前端依赖

```bash
cd bjt_agent/frontend
npm install
```

### 4.8 安装 RAG Memory 依赖

```bash
cd bjt_agent/rag_memory_service
npm install
```

### 4.9 配置环境变量

```bash
cd bjt_agent/backend

# 复制配置模板
cp .env.example .env

# 编辑 .env 文件（必须配置以下项）
nano .env
```

**必须配置项：**

```bash
# 数据库
DATABASE_URL="postgresql+asyncpg://bjt_user:你的强密码@localhost:5432/bjt_db"

# Redis
REDIS_URL="redis://localhost:6379/0"

# 安全密钥（生成随机字符串）
SECRET_KEY="your-super-secret-key-change-in-production-$(openssl rand -hex 32)"

# Mini-Max API（用于 LLM + 图像理解）
MINI_AGENT_API_KEY="你的MiniMax API Key"

# 智谱 AI（用于 Embedding）
ZHIPU_API_KEY="你的智谱 AI API Key"

# 代理（如需要）
HTTP_PROXY="http://127.0.0.1:7890"
HTTPS_PROXY="http://127.0.0.1:7890"
```

### 4.10 初始化数据库

```bash
# 创建 PostgreSQL 数据库和用户
sudo -u postgres psql << 'EOF'
CREATE DATABASE bjt_db;
CREATE USER bjt_user WITH ENCRYPTED PASSWORD '你的强密码';
GRANT ALL PRIVILEGES ON DATABASE bjt_db TO bjt_user;
\c bjt_db
GRANT ALL ON SCHEMA public TO bjt_user;
EOF
```

### 4.11 启动服务

```bash
# 使用管理脚本启动所有服务
cd bjt_agent
./scripts/bjt.sh start

# 检查服务状态
./scripts/bjt.sh status
```

**预期输出：**
```
[backend] Running (PID: xxxxx) ✓
[frontend] Running (PID: xxxxx) ✓
[rag_memory] Running (PID: xxxxx) ✓
[celery_review] Running (PID: xxxxx) ✓
[celery_parser] Running (PID: xxxxx) ✓
```

### 4.12 验证服务

```bash
# 健康检查
curl http://localhost:8000/health
curl http://localhost:3001/api/status

# 访问前端
# 浏览器打开 http://服务器IP:3000
```

## 5. 方式二：Docker Compose 部署（推荐生产环境）

### 5.1 安装 Docker

```bash
# 安装 Docker
curl -fsSL https://get.docker.com | sh

# 安装 Docker Compose
apt install -y docker-compose

# 添加当前用户到 docker 组（免 sudo）
sudo usermod -aG docker $USER
newgrp docker
```

### 5.2 配置环境变量

```bash
cd bjt_agent

# 创建 .env 文件
cat > .env << 'EOF'
# 数据库密码（强密码）
POSTGRES_PASSWORD=your-strong-postgres-password

# Backend
DATABASE_URL=postgresql+asyncpg://bjt_user:your-postgres-password@postgres:5432/bjt_db
REDIS_URL=redis://redis:6379/0
SECRET_KEY=your-super-secret-key-change-in-production
MINI_AGENT_API_KEY=your-minimax-api-key
MINI_AGENT_API_BASE=https://api.minimaxi.com
MINI_AGENT_MODEL=MiniMax-M2.7-highspeed
ZHIPU_API_KEY=your-zhipu-api-key
EOF
```

### 5.3 启动服务

```bash
cd bjt_agent

# 启动所有服务（包含 PostgreSQL 和 Redis）
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f backend
```

### 5.4 初始化数据库（首次）

```bash
# 进入后端容器执行数据库迁移
docker-compose exec backend bash
cd /app/backend
python -c "from database import engine; import asyncio; from sqlalchemy.ext.asyncio import create_async_engine; asyncio.run(create_async_engine('postgresql+asyncpg://bjt_user:your-postgres-password@postgres:5432/bjt_db').run_sync())"
# 或执行 Alembic 迁移（如果配置了）
# alembic upgrade head
exit
```

## 6. 配置管理

### 6.1 各服务配置位置

| 服务 | 配置文件 |
|------|----------|
| Backend | `backend/.env` |
| Frontend | `frontend/.env.production` |
| RAG Memory | `rag_memory_service/.env` |
| Nginx | `deploy/nginx/` |

### 6.2 配置优先级

环境变量 > `.env` 文件 > 代码默认值

### 6.3 关键配置项说明

#### Backend (`backend/.env`)

| 配置项 | 说明 | 示例 |
|--------|------|------|
| `DATABASE_URL` | PostgreSQL 连接串 | `postgresql+asyncpg://user:pass@host:5432/db` |
| `REDIS_URL` | Redis 连接串 | `redis://host:6379/0` |
| `SECRET_KEY` | JWT 签名密钥 | 随机 64 字符字符串 |
| `MINI_AGENT_API_KEY` | MiniMax API Key | `eyJh...` |
| `ZHIPU_API_KEY` | 智谱 AI API Key | `abcd...` |
| `WORKSPACE_DIR` | 文档存储目录 | `./workspace` |

#### Frontend (`frontend/.env.production`)

| 配置项 | 说明 | 示例 |
|--------|------|------|
| `VITE_API_BASE` | 后端 API 地址 | `/api`（通过 Nginx）或 `http://localhost:8000/api` |

#### RAG Memory (`rag_memory_service/.env`)

| 配置项 | 说明 | 示例 |
|--------|------|------|
| `MINI_AGENT_API_KEY` | MiniMax API Key | 同 Backend |
| `ZHIPU_API_KEY` | 智谱 AI API Key | 同 Backend |
| `PORT` | 服务端口 | `3001` |
| `DOCUMENTS_PATH` | 知识库文档路径 | `./knowledge_docs` |
| `EMBEDDING_MODEL` | Embedding 模型 | `embedding-2` 或 `embedding-3` |

### 6.4 HTTPS 配置

生产环境必须启用 HTTPS（浏览器 SSE 要求安全上下文）。

**使用 Let's Encrypt 证书：**

```bash
# 安装 certbot
sudo apt install -y certbot python3-certbot-nginx

# 获取证书（假设域名已解析）
sudo certbot --nginx -d your-domain.com

# 自动续期测试
sudo certbot renew --dry-run
```

## 7. 服务管理

### 7.1 使用管理脚本

```bash
# 启动所有服务
./scripts/bjt.sh start

# 停止所有服务
./scripts/bjt.sh stop

# 查看服务状态
./scripts/bjt.sh status

# 重启所有服务
./scripts/bjt.sh restart
```

### 7.2 手动管理（调试用）

```bash
# 激活 Python 环境
conda activate py311

# 启动后端
cd backend
python -m uvicorn main:app --host 0.0.0.0 --port 8000 --workers 4

# 启动 Celery Worker（review 队列）
celery -A celery_app worker --loglevel=info --concurrency=2 -Q review

# 启动 Celery Worker（parser 队列）
celery -A celery_app worker --loglevel=info --concurrency=2 -Q parser --max-memory-per-child=2000000

# 启动前端
cd frontend
npm run dev

# 启动 RAG Memory
cd rag_memory_service
npm run dev
```

### 7.3 日志位置

| 服务 | 日志文件 |
|------|----------|
| Backend | `scripts/logs/backend.log` |
| Frontend | `scripts/logs/frontend.log` |
| Celery Review | `scripts/logs/celery_review.log` |
| Celery Parser | `scripts/logs/celery_parser.log` |
| RAG Memory | `scripts/logs/rag_memory.log` |

```bash
# 实时查看日志
tail -f scripts/logs/backend.log

# 查看错误日志
grep -i error scripts/logs/backend.log
```

### 7.4 Nginx 配置

```bash
# 复制 Nginx 配置
sudo cp deploy/nginx/bjt /etc/nginx/sites-available/bjt
sudo ln -s /etc/nginx/sites-available/bjt /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重载 Nginx
sudo systemctl reload nginx
```

## 9. 常见问题排查

### 9.1 服务启动失败

**检查端口占用：**
```bash
lsof -i :8000   # Backend
lsof -i :3000   # Frontend
lsof -i :3001   # RAG Memory
```

**检查日志：**
```bash
tail -50 scripts/logs/backend.log
```

### 9.2 数据库连接失败

```bash
# 测试 PostgreSQL 连接
psql -h localhost -U bjt_user -d bjt_db

# 检查连接串格式
# postgresql+asyncpg://user:password@host:port/database
```

### 9.3 Redis 连接失败

```bash
# 测试 Redis 连接
redis-cli ping

# 清除可能的阻塞数据
./scripts/bjt.sh stop
# 然后手动清理 Redis
```

### 9.4 Celery Worker 不执行任务

```bash
# 检查 Worker 是否正常运行
ps aux | grep celery

# 查看 Worker 日志
tail -f scripts/logs/celery_review.log
tail -f scripts/logs/celery_parser.log

# 手动触发任务测试
cd backend
python -c "from tasks.review_task import process_review; process_review.delay('test-id')"
```

### 9.5 API Key 无效

```bash
# 测试 Mini-Max API
curl -H "Authorization: Bearer $MINI_AGENT_API_KEY" https://api.minimaxi.com/v1/models

# 测试智谱 AI
curl -X POST https://open.bigmodel.cn/api/paas/v4/embeddings \
  -H "Authorization: Bearer $ZHIPU_API_KEY" \
  -d '{"model":"embedding-2","input":"test"}'
```

### 9.6 前端无法连接后端

```bash
# 检查 Nginx 配置
sudo nginx -t

# 检查代理是否生效
curl -I http://localhost:3000/api/health
```

## 10. 数据备份

### 10.1 PostgreSQL 备份

```bash
# 创建备份脚本
cat > ~/backup_db.sh << 'EOF'
#!/bin/bash
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR=~/backups
mkdir -p $BACKUP_DIR

# 备份数据库
pg_dump -h localhost -U bjt_user bjt_db > $BACKUP_DIR/bjt_db_$DATE.sql

# 保留最近 30 天备份
find $BACKUP_DIR -name "bjt_db_*.sql" -mtime +30 -delete

echo "Backup completed: bjt_db_$DATE.sql"
EOF

chmod +x ~/backup_db.sh

# 添加到 crontab（每天凌晨 3 点执行）
echo "0 3 * * * ~/backup_db.sh" | sudo tee -a /var/spool/cron/crontabs/root
```

### 10.2 Redis 备份

```bash
# Redis 数据自动持久化到 RDB 文件
# 配置文件位于 docker-compose 卷挂载目录

# 手动备份
redis-cli SAVE
cp /var/lib/redis/dump.rdb ~/backups/redis_$(date +%Y%m%d).rdb
```

### 10.3 工作目录备份

```bash
# 备份用户上传的文档和生成的内容
tar -czf ~/backups/workspace_$(date +%Y%m%d).tar.gz \
  --exclude='workspace/**/node_modules' \
  --exclude='workspace/**/.git' \
  ~/bjt_agent/workspace
```
