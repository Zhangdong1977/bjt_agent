# 前置准备工作

## 需要大模型具备的常识
### 常识1
业绩合同包含两类，一类是单项合同，另一类是框架合同。如招标文件中没有明确说明，通常对于业绩的计算，单项合同是按合同金额计算，而框架合同是以提供的该框架合同下的所有订单金额总和计算。而且，只有按要求提供了证明材料才会被认可，未提供或提供不完整的，均不会被计算在内。
### 常识2
业绩是否满足时间范围要求，通常是指业绩合同的签订时间是否在规定的时间范围内。
### 常识3
如果招标要求中对业绩时间范围的要求是截止到投标截止日，那们需要在招标文件上下文中找到投标截止日，替换为具体日期。
### 常识4
对业绩的证明材料的检查方法是：
- 单项合同：
    - 合同的乙方名称应与本次投标方名称应该一致
    - 合同的签订时间应该在业绩要求的范围之内
    - 合同金额即是有效业绩金额
    - 发票的开票方应与本次投标方名称一致
    - 发票中应有体现对应的项目名称，该名称要与对应的单项合同中的项目名称一致。
    - 发票查询截图中的发票号码、开票金额、等信息要与发票上的信息一致。
    - 单项合同落款处要有甲乙双方签章，签章的名称要和甲乙双方的名称一致。
- 框架合同：
    - 框架合同的乙方名称应与本次投标方名称应该一致
    - 框架合同的签订时间应该在业绩要求的范围之内
    - 框架合同下所有订单的金额总和即是有效业绩金额
    - 每个订单对应的发票的开票方应与本次投标方名称一致
    - 每个订单对应的发票中应有体现对应的项目名称，该名称要与对应的框架合同中的项目名称一致。
    - 每个订单对应的发票查询截图中的发票号码、开票金额、等信息要与发票上的信息一致。
    - 框架合同落款处应有甲乙双方的签章，签章的名称要和甲乙双方的名称一致。
    - 订单中应有甲乙双方的签章，签章的名称要和甲乙双方的名称一致。

以上检查点有任意一项不满足，对应的业绩均不能算作投标总业绩之内。
### 尝试5
有的招标文件中，会把业绩作为资格项，比如，要求至少500万以上的同类型业绩，或者要求至少有一个相关业绩。此时，不止要检查业绩得分情况，还要判断知否满足资格项要求。

# 响应文件检查项定义
## 检查项1
### 检查项名称
检查有效业绩总额并打分
### 检查项规则描述
首先找到投标文件中业绩材料的章节，根据该章节的内容，检查是否提供了所有的业绩材料。
1. 识别该章节中业绩汇总表中的信息，了解有哪些业绩。
2. 根据前置准备工作中常识4的对业绩的证明材料的检查方法的描述，对盖章接下的每一条业绩进行逐个检查，判断业绩是否有效，并和上一步汇总表中的业绩比对，确认是否一致。如果业绩有效，就标记为有效业绩，否则标记为无效业绩。
3. 给出有效业绩的清单及总额，根据总额，按照招标文件中对业绩的要求描述，给出对应的得分。如果提供的业绩材料中有无效的业绩，也需要在清单中进行标记，并给出判定无效的原因。
### 正例 
在投标文件中找到了业绩章节，并前置准备工作中常识4的对业绩的证明材料的检查方法的描述，逐条对业绩进行了检查。有效业绩的清单及总额为：
|序号|合同名称|合同金额（万元）|是否有效|无效原因|
|--|--|--|--|--|
|1|合同A|100|是||
|2|合同B|200|否|未提供发票|
|3|合同C|300|是||

有效业绩总额为400万元。
根据招标文件中对业绩的要求描述，得分为3分。
