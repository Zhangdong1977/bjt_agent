# 系统上线前准备清单

本文档整理标书审查智能体系统上线前所需的外部依赖准备工作。

---

## 一、智谱AI（Embedding 服务）

| 项目 | 说明 |
|------|------|
| **平台** | 智谱AI开放平台（zhipuai.cn） |
| **功能用途** | 文档 Embedding 向量化（用于 RAG 检索） |
| **操作步骤** | 1. 访问 https://zhipuai.cn 注册公司账户<br>2. 完成企业实名认证<br>3. 进入控制台 → 找到 Embedding 服务<br>4. 充值 **100 元**<br>5. 获取 API Key 并配置到 `backend/config.py` 的 `ZHIPU_API_KEY` |
| **配置项** | `ZHIPU_API_KEY`、`ZHIPU_EMBEDDING_MODEL`（如使用 `embedding-2`） |

---

## 二、Minimax（LLM + 图像理解）

| 项目 | 说明 |
|------|------|
| **平台** | MiniMax开放平台（minimaxi.com） |
| **功能用途** | BidReviewAgent 核心推理 + 图像理解（文档解析） |
| **操作步骤** | 1. 访问 https://api.minimaxi.com 注册公司账户<br>2. 完成企业认证<br>3. 订阅 **Max-极速版**（高速推理通道）<br>4. 获取 API Key 并配置到 `backend/config.py` 的 `MINIMAX_API_KEY` 和 `MINIMAX_API_BASE` |
| **当前配置** | 模型：`MiniMax-M2.7-highspeed`（已在代码中） |
| **备注** | Mini-Agent 已通过 submodule 集成，只需确保 API Key 有效 |

---

## 三、云服务器

| 项目 | 说明 |
|------|------|
| **规格** | **4C16G**（4核CPU / 16GB内存） |
| **系统** | 推荐 Ubuntu 22.04 LTS 或 CentOS 8 |
| **用途** | 运行 Backend (FastAPI) + Frontend (Vue3) + Celery Workers + Redis + PostgreSQL |
| **所需端口** | 22（SSH）、3000（前端）、8000（后端API）、6379（Redis，对内）、5432（PostgreSQL，对内） |
| **存储** | 建议 **100GB+ SSD** 用于文档存储和工作空间 |
| **建议配置** | - 系统用户（非 root）<br>- Docker / Docker Compose（推荐）或直接 conda 环境<br>- Nginx 反向代理（可选，用于前端静态资源和服务聚合） |
| **参考 IP** | 当前测试环境：`183.66.37.186:7004`（PostgreSQL）、`:7005`（Redis）— 正式环境需替换 |

---

## 四、域名 + SSL 证书

| 项目 | 说明 |
|------|------|
| **域名** | 需要注册一个可用的域名，解析到云服务器公网 IP |
| **SSL 证书** | 推荐使用 **Let's Encrypt** 免费证书（certbot 自动续期）<br>也可使用云厂商提供的付费证书 |
| **配置位置** | Nginx 或 FastAPI Uvicorn 配置 HTTPS<br>参考：`backend/config.py` 中的 `DOMAIN` 和 `SSL_KEY_FILE` / `SSL_CERT_FILE` |
| **HTTPS 要求** | SSE 连接必须使用 HTTPS，否则浏览器 EventSource 无法工作 |

---

## 五、服务依赖汇总

| 服务 | 用途 | 配置变量 |
|------|------|---------|
| 智谱AI | Embedding | `ZHIPU_API_KEY` |

| PostgreSQL | 数据持久化 | `DATABASE_URL`（当前 `183.66.37.186:7004`） |
| Redis | Celery Broker + SSE | `REDIS_URL`（当前 `183.66.37.186:7005`） |
| 云服务器 | 全部服务运行载体 | — |

---

## 六、配置检查清单

上线前请确认以下配置项已正确填写：

- [ ] `ZHIPU_API_KEY` — 智谱 Embedding Key
- [ ] `MINIMAX_API_KEY` — Minimax API Key
- [ ] `DATABASE_URL` — 正式环境 PostgreSQL 地址
- [ ] `REDIS_URL` — 正式环境 Redis 地址
- [ ] `DOMAIN` — 系统访问域名
- [ ] `SSL_CERT_FILE` / `SSL_KEY_FILE` — HTTPS 证书路径
- [ ] `workspace` — 文档存储根目录（建议挂载大容量磁盘）
