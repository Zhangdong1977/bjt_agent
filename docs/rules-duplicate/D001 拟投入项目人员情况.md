# D001 拟投入项目人员情况

## 检查目标
检查 A、B 两份技术应标书中拟投入人员资料是否存在超出行业通用表述范围的高度一致内容。结论仅用于提示人工复核，不作串标、围标等法律定性。

## 重点检查项

### 检查项1：个人业绩描述
- 比对双方人员“主导项目经验”的核心动词、项目规模数字、技术难题及解决路径。
- 若人员姓名不同，但项目叙述结构、数字组合、罕见措辞和错别字均一致，应标记为疑似不合理重复。

### 检查项2：人员归属逻辑
- 检查社保缴纳单位、工作经历、内部项目代号与人员所属公司的逻辑关系。
- 若 A 方人员经历出现仅应属于 B 方的内部项目代号，且上下文表述一致，应重点标记。

### 检查项3：证书异常
- 比对证书名称、编号和图片识别文本。
- 若同一位置重复出现相同的非正常 OCR 错误，例如双方都把证书号中的“8”识别为“B”，应标记为 OCR 异常证据。

### 检查项4：人员承诺措辞
- 检查针对本项目的驻场、到岗和履职承诺。
- 若双方均采用非标准且高度个性化的表达，例如“余必亲临，昼夜督造”，应标记为疑似不合理重复。

## 合理重复
- 学历、学位的标准名称。
- 通用职业资格、职称和注册等级名称。
- 招标文件明确要求原文响应的人员配置条件。
- 法规或国家标准中的固定名称。

## 不合理重复
- 修改姓名后仍保留完全相同的项目业绩叙述、数字组合和错误。
- 跨公司出现相同内部代号、相同异常经历或相同证书识别错误。
- 双方独立形成概率较低的个性化承诺、文言式措辞或罕见错别字。

## 输出要求
- 必须引用 A、B 双方原文和位置。
- 必须使用工具给出的相似度。
- 必须说明为何属于合理重复或疑似不合理重复。
- 不得输出确定性的违法结论。
