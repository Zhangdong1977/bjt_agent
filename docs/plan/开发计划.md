## 阶段三：增强文档解析功能

### 需求背景
当前文档解析仅提取纯文本，招标书和投标书中包含丰富的结构化信息：
- **章节标题** - 文档导航
- **正文内容** - 技术规格、要求
- **列表项** - 检查项、要求清单
- **表格** - 报价表、技术参数表
- **图片** - 流程图、示意图

### 技术方案

#### 1. LibreOffice HTML 模式提取
```bash
libreoffice --headless --convert-to html --outdir <输出目录> <文件>
```

#### 2. HTML 解析提取结构化内容
使用 `BeautifulSoup4` 解析 HTML：
- **章节/标题**: `<h1>` - `<h6>` 标签
- **正文**: `<p>` 标签
- **列表**: `<ul>`, `<ol>`, `<li>` 标签
- **表格**: `<table>`, `<tr>`, `<td>`, `<th>` 标签
- **图片**: `<img>` 标签的 `src` 属性

#### 3. 转换为 Markdown 格式
- 标题 → `#`, `##`, `###`
- 列表 → `-`, `1.`
- 表格 → Markdown 表格格式
- 图片 → `![alt](path)`

#### 4. 图片 LLM 分析
使用 MiniMax Vision API 分析图片内容
