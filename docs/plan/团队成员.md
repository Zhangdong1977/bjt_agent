为了高效、高质量地完成该项目，组建一个 4-5人的小规模敏捷开发团队。角色配置如下：

1. 后端开发工程师 (Backend Engineer) - 2人
这是项目的主力，负责所有非前端逻辑。由于涉及FastAPI、Celery、数据库、Agent等多个模块，建议配置2人分工协作。

核心职责：

人员A (API & 服务层)：负责FastAPI应用、API路由、认证、数据模型（SQLAlchemy）、Service层业务逻辑、与前端联调。**当前重点：结构化文档解析框架搭建 + 图片 LLM 分析集成**

人员B (Agent & 任务层)：负责 HTML 解析器实现、Markdown 转换器、结构化内容提取（章节/列表/表格/图片）、与 Mini-Agent 的 Tool 集成。

关键技能：

必须：Python (精通FastAPI, SQLAlchemy), PostgreSQL, Redis, 异步编程 (asyncio)。

重要：熟悉Celery、熟悉LLM应用开发（Prompt Engineering）、熟悉HTML解析（BeautifulSoup4）。

2. 前端开发工程师 (Frontend Engineer) - 1人
负责实现直观、交互良好的用户界面。

核心职责：

搭建Vue3项目（Vite, Pinia, Vue Router）。

实现所有页面和组件，特别是上传组件、实时审查时间线组件（处理SSE流式数据）、结果展示表格。

对接后端API，处理文件上传、任务状态轮询等。

关键技能：

必须：Vue3, TypeScript, Pinia, 熟悉SSE (Server-Sent Events) 或 WebSocket。

重要：UI框架 (如Element Plus, Ant Design Vue)、处理文件上传和下载。

3. 测试/质量保证工程师 (QA Engineer) - 1人
由于系统涉及LLM，其输出具有不确定性，需要专门的测试来保证系统的稳定性和结果的准确性。

核心职责：

编写测试用例，覆盖正常流程、异常情况（如文档损坏、RAG服务不可用）、并发场景。

重点测试：Agent对不同标书（格式、内容、长度）的审查准确性，以及严重程度判断的合理性。

使用Chrome devtool MCP，在VNC的DISPLAY=:2的显示桌面上进行集成测试和端到端（E2E）测试，确保各模块协作无误。

**当前任务：协助验证结构化解析功能，编写端到端测试用例**

关键技能：

必须：熟悉测试流程、能编写清晰可复现的Bug报告。

重要：了解Python或JavaScript自动化测试框架（如pytest, Selenium/Playwright）、对AI应用测试有初步概念。

4. 项目经理/技术负责人 (PM/Tech Lead) - 1人 (可由核心开发兼任)
负责项目的整体推进、风险控制和关键技术决策。如果团队经验不足，此角色尤为重要。

核心职责：

分解需求，制定详细的迭代计划（如按Phase划分）。

协调团队内部和后端两人之间的工作，确保接口和模型定义统一。

负责与需求方（或自己作为需求方）沟通，澄清模糊点。

把控关键技术难点，如Mini-Agent的集成方式、RAG查询效果优化、并发控制策略等。

负责代码审查，确保代码质量和架构一致性。
