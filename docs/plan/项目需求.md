# 标书审查智能体系统 - 项目需求文档

## 1. 项目概述

### 1.1 项目目标

通过对 Mini-Agent 项目进行二次开发，实现一个**标书审查智能体**系统。该系统能够：
- 查询招标书获得招标要求
- 查询企业知识库获得企业内部知识
- 对比应标书与招标要求，指出不符合项（位置、严重程度、改进建议）

### 1.2 技术栈

| 组件 | 技术选型 |
|------|---------|
| 后端框架 | Python + FastAPI |
| 智能体基础 | Mini-Agent（git子模块，ReAct风格） |
| 知识库服务 | rag_memory_service（Express.js） |
| 任务队列 | Celery（Redis broker） |
| 数据库 | PostgreSQL |
| 前端框架 | Vue3 |
| 运行环境 | Windows / Linux |
| LLM模型 | MiniMax-M2.7-highspeed |

### 3.1 数据库表结构

```sql
-- 用户表
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    username VARCHAR(100) UNIQUE NOT NULL,
    email VARCHAR(255) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 项目表（一个审查任务对应一个项目）
CREATE TABLE projects (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    name VARCHAR(255) NOT NULL,
    description TEXT,
    status VARCHAR(50) DEFAULT 'draft',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 文档表
CREATE TABLE documents (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    project_id UUID NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
    doc_type VARCHAR(50) NOT NULL,  -- 'tender' (招标书) 或 'bid' (应标书)
    original_filename VARCHAR(255) NOT NULL,
    file_path VARCHAR(500) NOT NULL,
    parsed_md_path VARCHAR(500),
    parsed_images_dir VARCHAR(500),
    page_count INTEGER,
    word_count INTEGER,
    status VARCHAR(50) DEFAULT 'pending',  -- pending, parsing, parsed, failed
    parse_error TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 审查任务表
CREATE TABLE review_tasks (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    project_id UUID NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
    status VARCHAR(50) DEFAULT 'pending',
    celery_task_id VARCHAR(255),
    started_at TIMESTAMP,
    completed_at TIMESTAMP,
    error_message TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 审查结果表
CREATE TABLE review_results (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    task_id UUID NOT NULL REFERENCES review_tasks(id) ON DELETE CASCADE,
    requirement_key VARCHAR(255) NOT NULL,
    requirement_content TEXT NOT NULL,
    bid_content TEXT,
    is_compliant BOOLEAN DEFAULT FALSE,
    severity VARCHAR(50) NOT NULL,  -- critical, major, minor
    location_page INTEGER,
    location_line INTEGER,
    suggestion TEXT,
    explanation TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Agent执行步骤表（用于前端时间线）
CREATE TABLE agent_steps (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    task_id UUID NOT NULL REFERENCES review_tasks(id) ON DELETE CASCADE,
    step_number INTEGER NOT NULL,
    step_type VARCHAR(50) NOT NULL,  -- thought, tool_call, tool_result, final
    content TEXT NOT NULL,
    tool_name VARCHAR(100),
    tool_args JSONB,
    tool_result JSONB,
    duration_ms INTEGER,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 索引
CREATE INDEX idx_projects_user_id ON projects(user_id);
CREATE INDEX idx_documents_project_id ON documents(project_id);
CREATE INDEX idx_review_tasks_project_id ON review_tasks(project_id);
CREATE INDEX idx_review_results_task_id ON review_results(task_id);
CREATE INDEX idx_agent_steps_task_id ON agent_steps(task_id);
```

## 5. 核心Agent工具设计

### 5.1 RAGSearchTool - 企业知识库查询

```python
class RAGSearchTool(Tool):
    """从企业知识库检索相关信息"""

    name = "rag_search"
    description = "Search enterprise knowledge base for relevant information"

    async def execute(self, query: str, limit: int = 5) -> ToolResult:
        # 调用 rag_memory_service /api/search
        response = await httpx.post(
            "http://localhost:8001/api/search",
            json={"query": query, "limit": limit},
            timeout=30.0
        )
        if response.status_code == 200:
            data = response.json()
            results = data.get("results", [])
            snippet = "\n".join([r["snippet"] for r in results])
            return ToolResult(success=True, content=snippet)
        return ToolResult(success=False, error=f"RAG service error: {response.status_code}")
```

### 5.2 DocSearchTool - 招标书查询

```python
class DocSearchTool(Tool):
    """读取和搜索已解析的招标文件"""

    name = "search_tender_doc"
    description = "Search and read tender document content"

    async def execute(self, project_id: str, query: str = None) -> ToolResult:
        doc_path = Path(workspace) / project_id / "tender_parsed.md"
        content = doc_path.read_text(encoding="utf-8")
        if query:
            # 简单关键词搜索
            lines = content.split("\n")
            matches = [l for l in lines if query.lower() in l.lower()]
            content = "\n".join(matches[:20])
        return ToolResult(success=True, content=content)
```

### 5.3 ComparatorTool - 比对工具

```python
class ComparatorTool(Tool):
    """比对招标要求与应标书内容"""

    name = "compare_bid"

    async def execute(self,

        # 返回比对结果
        return ToolResult(success=True, content=json.dumps({
            "requirement": requirement,
            "bid_content": bid_content,
            "is_compliant": True/False,
            "severity": severity,
            "suggestion": "..."
        }))
```

## 6. 标书审查Agent流程

### 6.1 System Prompt

```
You are a professional tender/bid review agent.

## Your Task
Review a bid document against a tender document and enterprise knowledge base.

## Workflow
1. First, read the tender document (tender_parsed.md) to extract ALL requirements
2. For each requirement, query the enterprise knowledge base (rag_search) for relevant policies
3. Search the bid document (bid_parsed.md) for corresponding responses
4. Compare each requirement against the bid response
5. Identify non-compliant items with severity levels

## Output Format for Each Finding
- Requirement: [招标要求原文]
- Bid Content: [应标对应内容或"N/A"]
- Compliant: [Yes/No]
- Severity: [Critical/Major/Minor]
- Location: Page [X], Line [Y]
- Suggestion: [改进建议]

## Severity Guidelines
- Critical: Missing required documents, major compliance issues
- Major: Minor compliance gaps, incomplete responses
- Minor: Quality improvements, best practices

Be thorough and precise. Check every requirement systematically.
```

### 6.2 执行流程

```
1. Agent读取 tender_parsed.md
2. Agent使用 DocSearchTool 提取招标要求
3. 对于每个要求，Agent使用 RAGSearchTool 查询企业知识库
4. Agent读取 bid_parsed.md
5. Agent使用 ComparatorTool 比对每项要求
6. Agent汇总所有不符合项，按严重程度排序输出
```

## 8. 并发控制

### 8.1 Celery配置

```python
# 最大4并发任务
worker_concurrency = 4
app.conf.worker_prefetch_multiplier = 1

task_routes = {
    "backend.tasks.review_tasks.run_review": {"queue": "review"},
    "backend.tasks.document_parser.parse_document": {"queue": "parser"},
}
```

### 8.2 Redis并发计数

```python
# 任务启动前检查
current = int(redis.get('active_tasks:count') or 0)
if current >= 4:
    raise Exception("Maximum concurrent tasks reached")

# 任务启动时+1，结束时-1
redis.incr('active_tasks:count')
```

## 9. 文档解析策略

### 9.1 PDF解析

- 使用 PyMuPDF (fitz) 提取文本
- 使用 pdf2image 提取图片
- 图片保存到 `images/` 目录
- 文本保存为 `content.md`

### 9.2 Word解析

- 使用 python-docx 提取文本
- 使用 python-docx 的 inline shapes 提取图片
- 图片保存到 `images/` 目录
- 文本保存为 `content.md`

### 9.3 图片理解

- 使用 MiniMax 图像理解 API
- 解析后的内容追加到 `content.md`

## 12. 部署脚本

### 12.1 Windows启动脚本

```batch
@echo off
cd /d %~dp0

echo Starting Bid Review Agent Backend...
start "Backend" cmd /k "cd backend && python -m uvicorn main:app --reload --port 8000"

echo Starting Celery Worker...
start "Celery" cmd /k "cd backend && celery -A celery_app worker --loglevel=info"

echo Starting Frontend...
cd frontend && npm run dev
```

### 12.2 Linux启动脚本

```bash
#!/bin/bash
cd "$(dirname "$0")"

echo "Starting Bid Review Agent Backend..."
cd backend && nohup python -m uvicorn main:app --reload --port 8000 > backend.log 2>&1 &
BACKEND_PID=$!

echo "Starting Celery Worker..."
cd backend && nohup celery -A celery_app worker --loglevel=info > celery.log 2>&1 &
CELERY_PID=$!

echo "Starting Frontend..."
cd frontend && nohup npm run dev > frontend.log 2>&1 &
FRONTEND_PID=$!

echo "PIDs: Backend=$BACKEND_PID, Celery=$CELERY_PID, Frontend=$FRONTEND_PID"
```
